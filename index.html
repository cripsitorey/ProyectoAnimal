El código fuente ha sido desarrollado para ser ejecutado en un servidor local, como se evidencia a continuación, accesible mediante `127.0.0.1` o `localhost` (Recursos obtenidos de: W3Schools, 2024; Python Software Foundation, 2024).

1. Terminal con Servidor de Windows Activo:

Se muestra la ejecución del comando `Windows -m http.server 8000` en la terminal.

2. Navegador Accediendo a localhost:8000:

PARTE 3: Parámetros de la BIOS/UEFI

1. Foto de la BIOS o UEFI (Evidencia):

La BIOS/UEFI es el firmware fundamental que inicializa y prueba los componentes del hardware (Techopedia, 2023).

2. Explicación de Tres Parámetros:

  • Parámetro 1: [Boot Configuration ]

    Explicación: [La configuración de arranque (Boot Configuration) permite definir el orden de los dispositivos desde los cuales el sistema intentará iniciar, como discos duros, SSD, unidades USB o DVD. Su funcion es: garantizar que el sistema operativo se cargue desde la unidad correcta. ].

    Fuente: [Mundobytes, 2025]

  • Parámetro 2: [Fast Boot]

    Explicación: [Fast Boot acelera el inicio del equipo al omitir parte del proceso de verificación del hardware (POST). Esto reduce el tiempo de arranque pero puede dificultar el acceso al BIOS si se activa.Función principal: optimizar el tiempo de encendido reduciendo las comprobaciones iniciales.].

    Fuente: [HardZone, 2022]

  • Parámetro 3: [Boost Mode Select]

    Explicación: [Boost Mode Select o Game Boost permite aumentar automáticamente el rendimiento del procesador mediante overclocking preconfigurado. Este modo ajusta la frecuencia y el voltaje del CPU para mejorar la velocidad del sistema, aunque puede elevar la temperatura.Función principal: mejorar el rendimiento general mediante overclock automático.].

    Fuente: [wise cleaner,2025]
